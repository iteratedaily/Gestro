# Contributing

All contributions are welcome — bug reports, feature requests, documentation, and code.

## Reporting Bugs

Open an issue with:
- Browser version and OS
- Steps to reproduce
- Console errors (F12 → Console)

## Feature Requests

Open an issue describing:
- What you want to do
- Why existing functionality doesn't cover it
- Platform it applies to (YouTube, Instagram, TikTok, or new)

## Code Contributions

### Branch naming

- `feat/` for features
- `fix/` for bug fixes
- `docs/` for documentation

### Commit messages

Use conventional commits: `type(scope): description`.

Examples:
```
feat(adapter): add Netflix support
fix(recognizer): correct 2-finger detection threshold
docs(readme): update gesture table
```

### Pull request process

1. Fork the repo
2. Create a branch
3. Make your changes
4. Run `npm run build` — must succeed
5. Open a PR against `main`

The maintainer aims to review PRs within 48 hours.
